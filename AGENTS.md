## 项目概览

- 这是一个宠物洗护门店展示和预约页面项目。
- 项目名：`pet-care-next`。
- 技术栈：Next.js 15、React 19、TypeScript 5。
- 页面语言是中文，面向中国用户，主题是“暖爪宠物洗护店”。
- 主要功能是展示门店服务、套餐价格、护理流程、用户评价和预约表单。

## 运行命令

- 安装依赖：`npm install`
- 本地开发：`npm run dev`
- 生产构建：`npm run build`
- 生产启动：`npm run start`
- 代码检查：`npm run lint`

注意：`package.json` 里有 `npm run lint`，但当前项目没有单独看到 ESLint 配置文件。执行前要以实际命令结果为准。

## 目录结构

- `app/page.tsx`：首页主体内容，包含导航、首屏、服务、套餐、流程、评价和预约区域。
- `app/ArrivalTimeInput.tsx`：客户端组件，用来给预约时间输入框设置默认值。默认时间是“明天 09:30”。
- `app/layout.tsx`：Next.js 根布局，设置页面语言为 `zh-CN`，并配置页面标题和描述。
- `app/globals.css`：全局样式文件，控制整站颜色、排版、布局、响应式适配、轮播动画等。
- `public/assets/`：网页实际引用的静态图片目录，例如门店接待区、洗护区、美容区、地图图等。
- `assets/`：也保存了一份图片资源，和 `public/assets/` 里的图片重复；网页里真正通过 `/assets/...` 访问的是 `public/assets/`。
- `index.html`：一份静态 HTML 版本页面，看起来和 Next.js 页面内容相近，但当前 Next.js 项目的主入口是 `app/page.tsx`。
- `.next/`、`node_modules/`：生成目录和依赖目录，不要手动改。

## 页面内容

首页包含这些区域：

1. 顶部导航：品牌、服务、套餐、流程、预约入口。
2. 首屏介绍：门店定位、预约按钮、套餐按钮、关键数据和门店环境轮播图。
3. 服务卡片：基础香波洗护、造型修剪、皮毛深层护理、短时安心寄养。
4. 套餐价格：清爽洗护、精致美容、焕毛护理。
5. 护理流程：到店初检、方案确认、分区洗护、交付建议。
6. 用户评价：横向自动滚动的评价卡片。
7. 预约表单：主人姓名、联系电话、宠物类型、预约项目、期望到店时间、宠物情况。
8. 页脚：版权和品牌说明。

## 样式特点

- 主色是青绿色 `#1f9a9b`，辅助色是珊瑚橙 `#f47b65`。
- 整体视觉偏干净、温和、高端宠物护理空间。
- CSS 里大量使用 `grid` 和 `flex` 做布局：
  - `grid` 可以理解成“按几列几行摆东西”。
  - `flex` 可以理解成“让一排东西自动对齐和分散”。
- 卡片圆角基本是 `8px`，风格比较克制。
- 使用媒体查询适配手机和平板：
  - 920px 以下隐藏导航链接，布局从多列变少列。
  - 640px 以下大多变成单列，按钮变成满宽。
- 首屏环境图和评价区都有 CSS 动画，不依赖额外 JS。

## 开发注意事项

- 修改页面文案和结构时，优先改 `app/page.tsx`。
- 修改整体颜色、间距、布局、动画和移动端适配时，优先改 `app/globals.css`。
- 新增网页可访问图片时，应放到 `public/assets/`，然后用 `/assets/文件名` 引用。
- 不要直接改 `.next/` 里的文件，那是 Next.js 自动生成的。
- 预约表单当前没有提交逻辑，提交按钮是 `type="button"`，不会真正发送数据。
- 如果以后要实现真实预约提交，需要新增后端接口或接入表单服务，并处理手机号等隐私数据。
- 当前项目没有复杂状态管理，也没有数据库。

## 编码和语言

- 源码文件是 UTF-8，中文内容正常。
- 如果 PowerShell 里看到中文乱码，优先怀疑是终端编码显示问题，不要直接判断文件损坏。
- 新增或修改文档时继续使用中文，并保持结构清楚。
